Introdução

A API do Sistema de Postagem Automática fornece acesso programático às principais funcionalidades do sistema, permitindo integração com aplicativos de terceiros, automações e desenvolvimento de interfaces personalizadas.

Todas as respostas são fornecidas no formato JSON, e os métodos HTTP padrão (GET, POST, PUT e DELETE) são utilizados para interação com os recursos disponíveis.

URL Base

https://seudominio.com/api

Autenticação

A API utiliza autenticação baseada em token. Para acessar os endpoints protegidos, você precisa incluir o token de acesso no cabeçalho Authorization de todas as requisições.

Formato do Cabeçalho

Authorization: Bearer seu-token-aqui

Os tokens podem ser obtidos através do endpoint de login ou gerados manualmente na interface do sistema. Eles possuem validade limitada e podem ser revogados a qualquer momento.

POST /login

Autentica um usuário e retorna um token de acesso.

Parâmetros
Parâmetro Tipo Descrição
email * string Email do usuário
senha * string Senha do usuário
Exemplo de Requisição
POST /api/login
Content-Type: application/json

{
    "email": "usuario@exemplo.com",
    "senha": "senha123"
}
Resposta de Sucesso
200 OK 
{
    "status": "success",
    "message": "Login realizado com sucesso",
    "data": {
        "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
        "validade": "2025-06-19 00:56:28",
        "usuario": {
            "id": 1,
            "nome": "Usuário Teste"
        }
    }
}
Respostas de Erro
400 Bad Request 
{
    "status": "error",
    "message": "Email e senha são obrigatórios"
}
401 Unauthorized 
{
    "status": "error",
    "message": "Senha incorreta"
}
404 Not Found 
{
    "status": "error",
    "message": "Usuário não encontrado"
}
POST /register

Registra um novo usuário no sistema.

Parâmetros
Parâmetro Tipo Descrição
nome * string Nome completo do usuário
email * string Email do usuário
senha * string Senha (mínimo de 6 caracteres)
senha_confirmacao * string Confirmação da senha
Exemplo de Requisição
POST /api/register
Content-Type: application/json

{
    "nome": "Usuário Teste",
    "email": "usuario@exemplo.com",
    "senha": "senha123",
    "senha_confirmacao": "senha123"
}
Resposta de Sucesso
201 Created 
{
    "status": "success",
    "message": "Usuário registrado com sucesso",
    "data": {
        "id": 1,
        "nome": "Usuário Teste",
        "email": "usuario@exemplo.com"
    }
}
Respostas de Erro
400 Bad Request 
{
    "status": "error",
    "message": "As senhas não coincidem"
}
409 Conflict 
{
    "status": "error",
    "message": "Este email já está em uso"
}

POST /grupos

Cria um novo grupo.

Autenticação Requerida
Parâmetros
Parâmetro Tipo Descrição
nome * string Nome do grupo
url string URL do grupo no Facebook
facebook_id string ID do grupo no Facebook
descricao string Descrição do grupo
regras string Regras do grupo
ativo integer Status do grupo (0 = inativo, 1 = ativo). Padrão: 1
membros integer Quantidade de membros. Padrão: 0
privacidade string Tipo de privacidade (PUBLIC, CLOSED, SECRET). Padrão: UNKNOWN
ordem integer Ordem de exibição. Padrão: 0
Exemplo de Requisição
POST /api/grupos
Content-Type: application/json
Authorization: Bearer seu-token-aqui

{
    "nome": "Novo Grupo de Marketing",
    "url": "https://facebook.com/groups/987654321",
    "facebook_id": "987654321",
    "descricao": "Grupo para compartilhar dicas de marketing",
    "ativo": 1,
    "membros": 1500,
    "privacidade": "PUBLIC"
}
Resposta de Sucesso
201 Created 
{
    "status": "success",
    "message": "Grupo criado com sucesso",
    "data": {
        "id": 16,
        "nome": "Novo Grupo de Marketing"
    }
}
Resposta de Erro
400 Bad Request 
{
    "status": "error",
    "message": "O nome do grupo é obrigatório"
}

Postar

POST /postar

Realiza uma postagem imediata em um grupo.

Autenticação Requerida
Parâmetros
Parâmetro Tipo Descrição
grupo_id * integer ID do grupo onde será feita a postagem
campanha_id * integer ID da campanha de onde será extraído o anúncio
anuncio_id integer ID do anúncio específico a ser postado (opcional, se não informado, um anúncio será escolhido aleatoriamente da campanha)
Exemplo de Requisição
POST /api/postar
Content-Type: application/json
Authorization: Bearer seu-token-aqui

{
    "grupo_id": 1,
    "campanha_id": 5,
    "anuncio_id": 12
}
Resposta de Sucesso
200 OK 
{
    "status": "success",
    "message": "Publicação realizada com sucesso no grupo Grupo de Vendas",
    "data": {
        "post_id": "12345678_98765432",
        "grupo": "Grupo de Vendas",
        "anuncio": "Promoção de Verão"
    }
}
Respostas de Erro
400 Bad Request 
{
    "status": "error",
    "message": "Grupo e campanha são obrigatórios"
}
404 Not Found 
{
    "status": "error",
    "message": "Grupo não encontrado, inativo ou não pertence a este usuário"
}
500 Internal Server Error 
{
    "status": "error",
    "message": "Erro ao publicar no Facebook: (#200) Insufficient permission to post in this group"
}

Códigos de Erro

Código Status Descrição
200 OK Requisição processada com sucesso
201 Created Recurso criado com sucesso
400 Bad Request Requisição inválida ou mal formada
401 Unauthorized Autenticação necessária ou falha na autenticação
403 Forbidden Operação não permitida para o usuário autenticado
404 Not Found Recurso não encontrado
409 Conflict Conflito ao processar a requisição
422 Unprocessable Entity Falha na validação dos dados
429 Too Many Requests Limite de requisições excedido
500 Internal Server Error Erro interno do servidor
